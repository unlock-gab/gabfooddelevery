==================================================

1. a driver is assigned

- backend logic

- status transitions

- restaurant dashboard actions

==================================================

- DRAFT
- PENDING_DISPATCH
- DISPATCHING_DRIVER
- DRIVER_ASSIGNED
- AWAITING_CUSTOMER_CONFIRMATION
- NEEDS_UPDATE
- CONFIRMATION_FAILED
- CONFIRMED_FOR_PREPARATION
- PREPARING
- READY_FOR_PICKUP
- PICKED_UP
- ON_THE_WAY
- ARRIVING_SOON
- DELIVERED
- CANCELLED
- FAILED
- REFUNDED

Requirements:
- each status transition must be validated
- invalid transitions must be blocked
- each transition must create a history record
- each transition must record actor role and timestamp
- build reusable transition helpers

==================================================
CORE FLOW TO IMPLEMENT
==================================================

1. Customer places order
- create order
- create items
- calculate totals
- assign initial status: PENDING_DISPATCH

2. Restaurant receives order
- restaurant sees full order details
- restaurant sees preparation is locked
- no preparation action allowed yet

3. Dispatch engine runs
- choose candidate drivers based on:
  - same city / zone
  - close to restaurant
  - available
  - good performance
  - low workload
- create dispatch attempts
- send mission requests

4. Driver accepts mission
- first accepted driver wins
- lock assignment
- update order to DRIVER_ASSIGNED
- set AWAITING_CUSTOMER_CONFIRMATION

5. Driver confirms customer details
Driver can submit:
- confirmed
- needs correction
- failed confirmation

If confirmed:
- update order to CONFIRMED_FOR_PREPARATION

If needs correction:
- update order to NEEDS_UPDATE

If failed:
- update order to CONFIRMATION_FAILED

6. Restaurant starts preparation
- only possible after CONFIRMED_FOR_PREPARATION
- then PREPARING
- then READY_FOR_PICKUP

7. Driver picks up order
- PICKED_UP
- ON_THE_WAY
- ARRIVING_SOON

8. Delivery completion
- QR verification comes later in the next step
- but prepare status flow toward DELIVERED
