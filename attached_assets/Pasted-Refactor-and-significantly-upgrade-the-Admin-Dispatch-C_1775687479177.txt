Refactor and significantly upgrade the **Admin Dispatch Center** in the existing Tasty Crousty project.

Do not rebuild the entire project.
Keep the current architecture, business logic, and design language.
Focus specifically on building a premium and operationally powerful dispatch control center.

==================================================
GOAL
==================================================

The Dispatch Center must become the main control page for driver assignment operations.

It should allow admin/operators to:
- monitor all orders waiting for driver assignment
- understand current dispatch rounds
- see which drivers were notified
- see accept / reject / timeout responses
- detect blocked orders
- retry dispatch
- manually assign drivers
- understand dispatch efficiency by zone and time
- intervene quickly when automatic dispatch fails

This page must feel like a real live operations screen, not just a simple list.

==================================================
MAIN SECTIONS TO BUILD
==================================================

Build the Dispatch Center with these strong sections:

1. Top operational summary bar
2. Orders waiting for dispatch
3. Active dispatch rounds
4. Driver response monitoring
5. Failed / timed-out dispatches
6. Manual assignment tools
7. Dispatch analytics
8. Zone / wilaya dispatch pressure view
9. Activity feed
10. Filters and quick actions

==================================================

- orders waiting for dispatch

==================================================

- restaurant name
- customer zone

- wilaya

==================================================
